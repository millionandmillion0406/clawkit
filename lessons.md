# 五条铁规 · 来自 ZCode 的教训

> 2026-07-08 · 陈千安在与 ZCode 协作中总结

---

## 背景

Agent 在远程部署代码时，常见的失败模式是：用 sed 盲改服务器文件 → 连续崩溃 → 不知道哪处改坏了。

根本原因不是工具问题，是**缺乏渐进式验证的思维**——一口气改多处、一起部署、一起炸。

## 五条铁规

### 1. 改前先读
动手改服务器代码前，先 `cat` 全文读完。不要凭记忆 sed。

### 2. 改后验语法
每次修改后，立即跑 `python -m py_compile` 确认语法正确。

### 3. 部署验存活
部署到服务器后，`curl` 健康检查端点，HTTP 200 才算成功。

### 4. 一次一处
一次只改一个地方。验证通过后再改下一个。不可一口气改三处一起部署。

### 5. 每次留记录
不管成功还是失败，在知识管家或工作台文件中记录改了什么、怎么改的、结果。

## 真正的教训

ZCode 原话：
> "一口气改三处、一起部署、一起炸、不知道哪处错了。这不是技术问题，是思维模型缺失。"

**渐进式验证**——改一处→部署→验证→通过→改下一处——这是运维领域的核心思维模型。
